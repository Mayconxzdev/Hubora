Each file is a top-level React page component mounted under `src/pages/` and consumed by the app router. Data comes from two sources: the global Zustand store (`@/store/useStore`) for library entries and user profile, and feature-specific repositories (`@/services/localRepository`, `@/services/featureRepository`, `@/services/api`) for events, goals, and remote releases. UI is composed from shared primitives in `@/components/ui/*` (MediaCard, Button, Input, SEO) plus motion/react for transitions. Cross-cutting concerns are delegated outward — i18n via `useTranslation`, routing via react-router-dom, caching via @tanstack/react-query (Releases), and analytics/SEO via Helmet or the SEO component. There is no internal sub-package; each page owns its local state and filters, with derived data computed through `useMemo` over store slices.
