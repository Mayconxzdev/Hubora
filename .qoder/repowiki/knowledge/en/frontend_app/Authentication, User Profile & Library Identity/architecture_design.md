Two-layer split inside the scope:
- `src/services/` — pure service layer. `auth.ts` exposes an `authService` object wrapping Supabase Auth (`signInWithOAuth`, `signUp`, `resetPasswordForEmail`, `onAuthStateChange`) behind a `requireClient()` guard that throws when `VITE_SUPABASE_URL` / `VITE_SUPABASE_PUBLISHABLE_KEY` are missing; it maps raw Supabase users to the app's `AuthUser` shape. `identity.ts` is framework-free utility code providing provider-name normalization (`PROVIDER_PREFIXES` + `normalizeProvider`), canonical entry-id generation (`parseProviderIdentity` → `createEntryId`), work fingerprinting (`createWorkFingerprint`), text normalization, and `findLibraryEntry` matching against local library records.
- `src/pages/` — React page components. `Login.tsx`, `Register.tsx`, `ForgotPassword.tsx` are thin UI shells over `authService`, driven by `import.meta.env.VITE_ENABLE_EMAIL_LOGIN` and `VITE_ALLOW_PUBLIC_SIGNUP`; they share `AuthLayout`, `Helmet` titles, and `sonner` toast feedback. `Profile.tsx` reads/writes user data through `useStore().updateUser` and computes stats from the local library. `Connections.tsx` lazily imports `cytoscape` via dynamic `import('cytoscape')` to render a genre/creator graph of up to 120 library entries.
Dependency direction: pages → services → `@/lib/supabase` / `@/store/useStore` / `@/types`; services have no React dependencies, keeping them testable in isolation.
