Three sibling React page components share the same `hub-page` layout convention: `Settings.tsx` is the interactive hub — it reads/writes user preferences through the global `useStore`, calls `@/services/api` for health checks, delegates backup import/export to `@/services/backup` and `@/services/automaticBackup`, pins vault state via `@/services/vault`, and queries IndexedDB through `localRepository` / `featureRepository`. `Privacy.tsx` and `Terms.tsx` are pure presentational pages that render a fixed `sections` array into `hub-panel` cards and set SEO via `<SEO>` from `@/components/ui/SEO`. The leaf service `src/services/privacy.ts` owns the only in-process privacy boundary: `isAdultEntry`, `entriesSafeForSharing`, and `eventsSafeForSharing` filter out adult-vault or classified-as-adult items before they leave the device; it depends on `classifyAdult` from `@/services/adultPolicy` but is itself never imported by other services, keeping this module's dependency direction one-way (pages → store/services, not vice versa).
