Each file is a self-contained React page component (`Books`, `Manga`, `Comics`, `Novels`) following an identical layout: local state for `sort`/`genre`/`query` plus a debounced `debouncedQuery` (500 ms), a single `@tanstack/react-query` `useInfiniteQuery` keyed on `[<slug>-discover, sort, genre, debouncedQuery]` calling the matching `api.discover*` endpoint, and a `react-intersection-observer` `useInView` hook that triggers `fetchNextPage` when the sentinel element enters view. The UI is composed entirely from shared section components — `SectionPageLayout` (title/subtitle/error/skeleton footer) and `SectionToolbar` (search input, sort dropdown, genre dropdown) — so each page only supplies its own typed `*_GENRES` and `SORT_OPTIONS` arrays plus the data shape consumed by the layout. Dependency direction is one-way: pages → `@/services/api` + shared `@/components/section/*` + `@/hooks/useTranslation`; no cross-page imports exist.
